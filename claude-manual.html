CHAPTER 01

Claudeは「どんな存在」か

まず、Claudeが何のために作られているのかを理解しましょう。これを知らないまま使うと、「なぜうまくいかないのか」がわかりません。

🎯

設計の目的

「その会話の中で、人間にとって最も有益な回答を返す」ことに最適化されています。対話の瞬間に最善を尽くす存在です。

🤝

パートナーとして

「AIに仕事をさせる」のではなく「一緒に考えるパートナー」として設計されています。人間が方向を示し、Claudeが実行・整理・検証を担う分担が最も力を引き出します。

⚡

得意なこと

文書作成・整理・計算・図解・アイデア出し・構造化・翻訳。「人間が時間をかけてやっていた整理作業」を高速に代行できます。

⚠️

苦手なこと

自律的にルールを守り続けること・自分の出力を客観的に検証すること・長期的な記憶の維持。これは設計上の限界であり、人間が補う部分です。

ドリプロの考え方

「お互いのことを知って行くことがとても大切。」Claudeの限界を知った上でClaudeを設計することが、正しい活用の第一歩です。AIが答えを出すのではなく、人が自分で気づき、動けるようになるための道具として使っています。

CHAPTER 02

グループホームの業務で使える場面

「どんな時に使えるの？」という疑問に答えます。まずはこの中から1つ試してみてください。

📝
支援記録の草案作成
「〇〇さんが今日△△した」と伝えると、記録文の草案を作ってくれます。
📋
モニタリング記録
支援内容のポイントを箇条書きで伝えると、モニタリング文章を整理してくれます。
✉️
家族・関係者への連絡文
「〇〇を伝えたい、相手は利用者の母親」と伝えると、適切なトーンの文章を作ります。
📊
加算・制度の確認
「この加算を取るには何が必要？」と聞くと、要件を整理して教えてくれます。※必ず一次確認を。
📅
会議資料・議事録
話し合いの内容をメモで貼り付けると、議事録の形に整理してくれます。
🔍
わからない制度・言葉の説明
「〇〇ってどういう意味？」「この制度の概要を教えて」と気軽に聞けます。

スマホでも使えます

Claudeはスマホアプリ・ブラウザから使えるので、現場でそのまま活用できます。

CHAPTER 03

記憶の仕組みを理解する

Claudeの記憶は「人間の記憶」とまったく違います。この違いを知らずに使うと、「なぜ覚えていないの？」と感じます。

🔄

会話ごとにリセットされる

Claudeは新しい会話が始まるたびに、前の会話の内容をすべて忘れます。これは設計上の仕様です。「前に話したこと」を前提に新しい会話を始めると、Claudeは覚えていません。

設計上の仕様
🧠

「メモリ機能」で補える（Claude Chat）

Claude Chatには会話をまたいで情報を保持する「メモリ」機能があります。よく使う情報を記憶させておくと、毎回ゼロから説明しなくて済みます。

Chat活用ポイント
📄

「CLAUDE.md」で補える（Claude Code）

Claude Codeでは「CLAUDE.md」というファイルにルール・前提・背景を書いておくと、毎回その内容を読んで仕事を始めます。プロジェクトの「引き継ぎ書」として機能します。

Claude Code専用
⚡

同じ会話内なら覚えている

1つの会話の中では、最初から最後まで文脈を保持しています。長い作業は「1つの会話を続ける」のが基本。

注意点

よくある失敗パターン

  • 「前に話したこと」を前提に新しい会話を始める → Claudeは覚えていない
  • 重要なルールを口頭で伝えただけで終わる → 次の会話では消える
  • 「もうわかってるよね？」と省略する → 毎回伝え直すか、メモリに登録する
CHAPTER 04

Claude Chat と Claude Code の違い

ドリプロでは2種類のClaudeを使い分けています。

比較軸Claude ChatClaude Code
主な使い方会話・相談・資料作成・アイデア出しシステム開発・自動化・ファイル操作
誰が使うドリプロスタッフ全員が日常業務で活用主に開発担当（陽介さん・谷口・杉山）
アクセス方法ブラウザ・スマホアプリで即使えるClaudeデスクトップ版が主な使い方
記憶会話ごとにリセット（メモリ機能で補完）CLAUDE.mdで永続的に保持可能
できること文章生成・翻訳・計算・図解・制度確認・相談コード生成・ファイル操作・システム連携・自動実行
失敗したら「やり直して」で何度でもやり直せるGitHubで変更履歴が残るので元に戻せる

ドリプロの使い分けイメージ

  • Claude Chat = 現場でのアイデア出し・資料草案・制度確認・返信文作成
  • Claude Code = システム開発・書類自動生成・データ処理
  • 2つの連携 = Chatで設計してCodeで実装。Chatが「企画・指示役」でCodeが「実行部隊」
CHAPTER 05

人間が意識することで力が引き出せる

5つのことを意識するだけで、Claudeは最大の力を発揮します。

①

「何をしてほしいか」を明確に伝える

新人職員に「資料作って」と言っても「何の資料？」となりますよね。Claudeも同じです。目的・対象者・形式・ボリュームを伝えるほど、求めている答えに近づきます。

伝わりにくい例
「資料を作って」
伝わりやすい例
「グループホームの職員向けに、処遇改善加算の申請手順を5ステップで説明する資料を作って。読む人は加算の知識がほぼない想定で、A4一枚に収まる量で。」
②

チェックは「過程」を見る

Claudeは「答えを早く出す」方向に動きやすい特性があります。「確認しました」だけでなく、計算の過程が書かれているかを確認してください。

③

「なぜ？」を問い続ける

間違えたとき「直して」で終わらせず「なぜこうなったの？」と問うと、原因と対策が明確になり、次回の精度が上がります。

④

重要なことは「見える化」させる

計算シートを見せてもらう・チェック報告を宣言させるなど、プロセスを人間が確認できる形にすることが重要です。

⑤

うまくいった使い方・失敗した使い方を記録する

ルール・失敗パターン・成功パターンを書き残すことで、チーム全体の資産になります。

CHAPTER 08

よくある疑問

タップして回答を見てください。

失敗しても何も壊れません。Claude Chatは「やり直して」と言えば何度でも別の回答を出してくれます。使いながら覚えることが一番の近道です。

意図的な嘘はつきません。ただし、間違いを「正しい」と確信して出力することがあります。重要な制度情報は必ず一次ソースで確認してください。

Anthropicのプライバシーポリシーに従って管理されています。個人情報・機密情報の入力は最小限にしましょう。

最終判断は必ず人間がしてください。特に支援計画・契約・行政対応は必ず担当者が内容を確認・承認してから使用してください。

「指示をもっと具体的にする」を試してください。目的・対象・形式・ボリュームを加えるだけで大きく変わります。

安全性・プライバシーに関わる内容は意図的に断ります。別のアプローチを検討してください。「なぜできないか」を聞くと理由を教えてくれます。
