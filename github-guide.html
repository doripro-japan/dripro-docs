CHAPTER 01

GitHubのイメージ — Googleドキュメントとの比較

GitHubは、チームでコードを管理するためのサービスです。身近な「Googleドキュメント」との比較で理解しましょう。

比べるポイントGoogleドキュメントGitHub
保存場所クラウド上で直接編集する自分のPCで編集→アップロードする
同時編集リアルタイムで全員に反映されるアップロード（Push）した時に反映される
元に戻す「版の管理」で過去に戻れるセーブポイント（コミット）で戻れる
共有方法リンクを送る全員が同じ場所（main）を見る
一言でいうと同時に書ける共有ノート順番に書く共有ノート

ポイント

Googleドキュメントは「同時に書ける」。GitHubは「順番に書く」。
だから書く前に「今の最新版を見せて」と確認するのが大事、というだけの話です。

CHAPTER 02

用語の言い換え表 — こう言えばOK

Claude Codeに伝える時、どれでも通じます。自分が言いやすい言葉でOK。

操作こう言えばOK（どれでも通じます）何が起きる？
Pull（プル）「最新を取ってきて」「プルして」
「作業する準備して」「ダウンロードして」		GitHubから最新版を自分のPCに取得する
Push（プッシュ）「ホームページ更新して」「反映して」
「プッシュして」「アップロードして」		自分の変更をGitHubに送る → ホームページに自動反映
Commit（コミット）「セーブポイント作って」「保存して」
「コミットして」「記録して」		変更内容を記録する（まだアップロードはしない）
Fetch（フェッチ）「確認して」「チェックして」
「新しい変更ある？」		GitHubに新しい変更がないか確認する
コンフリクト「衝突」「ぶつかった」
「かぶった」		2人が同じ場所を変更した時に起きる

「最新を取ってきて」と言う
Claude Codeに言うだけでOKです。

実は、もっと簡単です

「ホームページ更新して」と言えば、Claude Codeが自動でPull（最新取得）してからPush（反映）します。つまり忘れても大丈夫。でも仕組みを知っておくと、トラブル時に安心です。

CHAPTER 04

なぜ「最新を取る」が必要？ — 共有冷蔵庫の例え

会社の共有冷蔵庫に「買い物リスト」が貼ってあるとイメージしてください。

✅ うまくいくパターン

① 朝、杉山さんがリストを見る →「牛乳・卵」と書いてある

② 杉山さんが「パン」を追記 → リストは「牛乳・卵・パン」

③ 午後、谷口さんがリストを見てから「チーズ」を追記

④ リストは「牛乳・卵・パン・チーズ」→ 全員の変更が残っている！

❌ 困るパターン

① 朝、杉山さんがリストを見る →「牛乳・卵」

② 朝、福冨さんもリストを見る →「牛乳・卵」

③ 杉山さんが「パン」を追記して貼り直す →「牛乳・卵・パン」

④ 福冨さんがリストを見ないまま「チーズ」を追記 →「牛乳・卵・チーズ」

→ 杉山さんの「パン」が消えた！

💡 解決策

④の前に、福冨さんが「今のリスト見せて」と確認する

→「牛乳・卵・パン」を確認してから「チーズ」を追記

→ リストは「牛乳・卵・パン・チーズ」✅

GitHubでの「今のリスト見せて」＝「最新を取ってきて」（Pull）です。

CHAPTER 06

実は自動でやってくれます

PullとPushを別々に覚える必要はありません。

「ホームページ更新して」と言えば
Claude Codeが自動で
Pull（最新取得）→ Push（反映）をやります
仕組みを知っておくと「なぜそうするのか」がわかるので、トラブル時に安心です。

じゃあ「最新を取ってきて」は何のため？

作業開始時に最新の状態で始められるためです。他の人が変更した後のコードで作業した方が効率がいいですし、無用な衝突を避けられます。忘れてもPush時に自動で取り込むので致命的ではありませんが、習慣にしておくのがおすすめです。

CHAPTER 07

衝突（コンフリクト）が起きたらどうする？

まず結論：ほぼ起きません。起きてもClaude Codeが解決してくれます。

ケース結果対応
別のファイルを変更問題なし ✅自動で合体
同じファイルの別の行を変更問題なし ✅自動で合体
同じファイルの同じ行を変更衝突 ⚠️どちらを残すか選択

現実的にどれくらい起きる？

ほぼ起きません。同じファイルの同じ行を同時に触ることは非常にまれです。万が一起きても、Claude Codeが「ここが衝突しています。どちらを残しますか？」と聞いてくれます。自分で解決する必要はありません。

CHAPTER 08

GitHub Desktopの使い方（確認用）

GitHub Desktopは「確認ツール」として使います。コードの変更はClaude Codeがやります。

使うボタンは2つだけ：

①
Fetch origin
「誰かがPushしたかな？」を確認する。新しい変更があれば「Pull origin」に変わる。
②
Pull origin
Fetch後に表示される場合に押す。最新版をダウンロード。Claude Codeの「最新を取ってきて」と同じ。

「Commit to main」「Push origin」は？

Claude Codeがやるので、GitHub Desktopからは基本的に使いません。GitHub Desktopは「確認する」ためのツールとして使ってください。

よくある疑問

Claude Codeが「ホームページ更新して」の時に自動でPullしてからPushするので、忘れても大丈夫です。

Claude Codeが「どちらを残しますか？」と聞いてくれます。指示してください。自分で解決する必要はありません。

「前の状態に戻して」とClaude Codeに言えば、いつでも元に戻せます。焦らなくて大丈夫です。

大丈夫です。別のファイルや別の行なら自動で合体されます。同じ行を触った場合のみClaude Codeが聞いてくれます。

開かなくても問題ありません。確認したい時だけ「Fetch origin」を押してください。
